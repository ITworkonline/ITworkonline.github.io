配置

OAuth 配置

必须与 Tesla 开发者平台中设置的 Redirect URI 完全一致（包括协议、域名、路径）

CORS 代理配置（解决 CORS 错误）

用于解决 CORS 错误的代理服务器 URL。如果遇到 CORS 错误，请填写此字段。你的代理服务器应该部署在 Vercel 或其他平台。

Fleet Telemetry 配置（必需）

部署在 Railway/Render 等平台的 Telemetry 服务器 URL（HTTP，不是 WebSocket）
车辆识别码，可在 Tesla 应用中查看，或查看前挡风玻璃左下角
用于配置车辆的 WebSocket URL（wss://），通常与服务器 URL 相同但使用 wss:// 协议
⚠️ 配置 Fleet Telemetry 必须通过 vehicle-command proxy。参考 vehicle-command 部署说明

更新设置

数据更新间隔，建议 2-10 秒。太短可能导致请求过于频繁。
